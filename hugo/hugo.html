Introduction du model OSI


Le modèle OSI (Open Systems Interconnection) est un cadre conceptuel qui définit les fonctions de communication d'un système de télécommunication ou d'un réseau informatique. Il a été développé par l'ISO (International Organization for Standardization) pour servir de référence universelle pour la conception et la compréhension des réseaux.

Le modèle OSI est divisé en sept couches, chacune représentant un ensemble spécifique de fonctions liées à la communication. De la couche physique à la couche application, voici les sept couches du modèle OSI :

Couche N°1

Couche Physique
(Physical Layer)

Couche N°1

Définit les caractéristiques physiques du support de transmission, comme les câbles, les signaux électriques, etc.

Couche N°2

Couche Liaison de Données
(Data Link Layer)

Couche N°2

Gère l'accès au support de transmission, détecte et corrige les erreurs au niveau physique.

Couche N°3

Couche Réseau
(Network Layer)

Couche N°3

Fournit des mécanismes de routage pour acheminer les paquets de données entre différents réseaux.

Couche N°4

Couche Transport
(Transport Layer)

Couche N°4

Assure le transport fiable des données d'une extrémité à l'autre, gère le contrôle de flux et la correction d'erreurs.

Couche N°5

Couche Session
(Session Layer)

Couche N°5

Établit, gère et termine les sessions de communication entre deux applications.

Couche N°6

Couche Présentation
(Presentation Layer)

Couche N°6

Gère la syntaxe de données, la compression, la cryptographie et la conversion des données pour assurer une communication entre systèmes hétérogènes.

Couche N°7

Couche Application
(Application Layer)

Couche N°7

Fournit des interfaces réseau pour les applications utilisateur, gère les services réseau tels que le courrier électronique, le transfert de fichiers, etc.
